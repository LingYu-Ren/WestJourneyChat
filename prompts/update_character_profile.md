## 更新规则

### 基础属性（词语列表）
- `description`：角色的身份背景描述（一句话，不超过 30 字）
- `personality`：性格特点，仅用词语，基于事件中的行为推断
- `traits`：外貌/习惯/标志性特征，仅用词语
- `abilities`：技能与能力，仅用词语
- `summary`：仅针对本次新增事件撰写 1~2 句增量叙述，描述角色在这些事件中的新进展或变化。**不要重复或改写已有 summary 的内容**，已有内容由系统自动保留。

**更新原则**：
- 已有词语保留，新词语追加，不得删除已有内容
- 词语不重复，不写完整句子

### 事件参与记录（`events` 列表）
对每个新增事件，提取该角色在此事件中的：

| 字段 | 说明 |
|------|------|
| `event_id` | 直接沿用输入中的 event_id |
| `event_name` | 直接沿用输入中的 title |
| `event_status` | 角色所处的情境描述：该事件发生时角色面临的外部环境、处境或形势（≤20字，如"群猴推举为王，众望所归"、"身陷天兵围困，形势危急"） |
| `character_decision` | 该角色在此事件中做出的关键决策或行动（≤15字） |
| `decision_result` | 该决策/行动带来的直接结果（≤15字，无明确结果填null） |

**规则**：
- 已有事件不重复添加（以 event_id 为唯一键）
- 只记录该角色**实际参与或被直接影响**的事件

## 输出格式
严格输出 JSON，不含任何解释：

```json
{
  "name": "{{CHARACTER_NAME}}",

  "description": "角色身份描述（≤30字）",
  "personality": ["性格词1", "性格词2"],
  "traits": ["特征词1", "特征词2"],
  "abilities": ["能力词1", "能力词2"],
  "summary": "仅本次事件的增量叙述（1~2句话，不含已有内容）",
  "events": [
    {
      "event_id": "ch001_001",
      "event_name": "事件标题",
      "event_status": "角色所处情境（≤20字，描述外部环境/处境/形势）",
      "character_decision": "角色决策（≤15字）",
      "decision_result": "决策结果（≤15字）"
    }
  ]
}
```
